Fixed the doli blanket-mute that was silently muting nothing. The `datadog_downtime_schedule.doli_mute` had `scope = "tenant:doli"`, but `scope` filters which evaluated metric *groups* are muted — and the doli monitors group by `kube_*` tags (kube_cluster_name/kube_namespace/kube_deployment), which carry no `tenant:doli` tag, so zero groups matched and `deployment_unavailable` kept re-triggering emails through the downtime. Changed to `scope = "*"` (mute all groups) while keeping `monitor_identifier.monitor_tags = ["tenant:doli"]` to restrict the mute to doli's monitors. Applied live 2026-07-24 (downtime replaced, since an in-progress downtime's start time can't be edited).
